      • This is applicable to outside the border.

    • Border
      • This is applicable to thickness of element.

    • Padding
      • This is the space between an element's content and its border.

  • Css Cascade

      • This is order of importance in which css is being apply to html.

    1. Position(This give priority to the last applied style)
      • p {color: red;
        color: blue; }

        [blue will override red because of the position.]

    2. Specificity(This is rank next to the position)
      • li {color: coral}
      • .first-class {color: red;}
      • li[draggable] {color: purple;}
      • #first-id {color: green}

        • If all these selectors are all apply to an element tag(li)
        The "id selector"(#first-id) will be apply because it
        will override the other selectors Because it is the most
        specific selector. follow by "attribute selector",
        "class selector", and "element selector".
        Note: Though some believe "attribute selector and
        class selector" are on the same level. But in most cases
        it do override class selector.

    3. Type(This is next after Specificity)
      1. <link rel="stylesheet" href="">
      2. <style></style>
      3. <h1 style="">hey</h1>

        4. External link is the last to be consider as it pave way for
        other type. internal is next to External css. The inline
        is the first and will override the rest type of css.

    4. importance(This is in the first level in the Cascade steps.)
      • p {color: red;}
      • p {color: green !important;}

        • In this instance, green will be apply because
        of the important notation. it will override any
        other styling available be it inline, id and so on.

    • Static Position

        This is the default position of element.
        No Properties like Top, right e.t.c
        are not applicable to this position

    • Relative Position

        This unlike static position is relative
        to its position. Properties like top,
        right e.t.c are applicable to this

    • Absolute Position

        This affect the flow of html element.
        This position is relative to its Parent
        element. Properties like top, left e.t.c
        are applicable to it. It can as well be
        control by z-index because it can appear
        on top of other element.

    • Fixed Position

        This like absolute position also affect flow
        of an html. It is fixed unlike other position.
        Properties like top, right e.t.c are applicable
        to it as well. This make element to remain fix
        when scrolling.

        Similar to fixed position

        div will occupy two columns and will be 100px. The rest

        of 20px.

So with the ratio as width, it makes it responsive.

  • Responsiveness In Details

    • Media Query

        Previously, we discuss about max-width. In media query, we also have min-width. for example:

      • @media(min-width:720px){
        h1 {font-size: 2rem}
        }
        • This means, when the screen size is at 720px,
        that is minimum of 720px and above. Increase the
        font size. This target bigger screen.

        Combining Breakpoint.

        We can combine breakpoint to target specific size eg:

      • @media(min-width:720px) and (max-width: 900px){
        ...
        }
        • This means, we want the effect on screen of range between
        720px and 900px. So anything bigger than 720px and less
        than 900px.

      • media(max-width:720px) and (min-width: 900px){
        ...
        }
        • Flipping it around make the interpretation to change. This
        means, we want changes below 720px on small screen and on
        bigger screen should be greater than 900px and higher.

        Device

        We can also see device usage. eg

      • @media screen(orientation: landscape) {
        ....
        }
        • We don't have to put 'screen' as it'll be applicable to all screen by default.
        Alternative to screen is print. This allow us to use media query to
        target only when our website is printed and to give it different layout.
        so we can either target screen or print and we don't have to be explicit
        by writing both the screen and the print out.

    • This is easy and neat modern tools. Which is simply implemented by creating a div that encapsulate the rest div's. Simple illustration is:

      .container { display: flex; gap: 10px; } p { background-color: darkseagreen; }

    Previously, we learned that div occupy full width while span is an inline element. p tag is also block element. img tag is an inline but with flexbox, all of the default behavior are overlook. And the width of each of these element inside flexbox will be based on the content size. So it'll try and let us have all our content on one line. We can then use Gap property to separate them. Default setting of a container(div) is like a block element(100% full width). We can also set it as: 'inline-flex'. Which is similar to the inline we already treated. It'll make flexbox occupy the space needed. Which means other things can also go and occupy in that same line. So we can either set the main container(div) as either 'flex' or 'inline-flex'.

    • Flex direction

        The default settings of flex is row. We can also set flex direction property to 'column.' When at default that is 'row'. the main axis tends to be on an horizontal line while the cross-axis will be on vertical line. Which when 'flex-basis' property is apply, it increases the width. If the flex-direction is set to 'column' the main axis will now be vertical and the cross-axis will now be the horizontal line. Which means when 'flex-basis' property is apply, it will increase the height.

      • .main { display: inline-flex; background-color: darkblue; border: solid rebeccapurple 5pt; gap: 120px; color: yellowgreen; font-size: 1.1rem; flex-direction: column; } .red { background-color: red; } .green { background-color: green; } .brown { background-color: brown; } .main > * { flex-basis: 80pt; }

        • In the above example, since the 'flex-direction' is column, flex-basis will increase the height of each items. If it is at default of row. The width will increase instead. We used 'inline-flex' to make the full-width not at default of block instead it should take the width of the content.

  • Flex-sizing[Shrinking and Growing]

      Flexbox has default behavior to adjust to content size but here we'll dive into full way to customize the behavior to our taste. In flexbox, we have a priority list which is range from 'content-width < width < flex-basis < max-width/min-width' It first look at whether max-width/min-width is set, then flex-basis, width, and finally the content-width. 'content-width' allows content to lie up next to each other. If shrink, it'll reduce to 'minimum-width' which is where the text wraps and they are stacked on top each other. The default behavior of flexbox is to shrink until it reach the minimum content size. The longest word determine the final size. On getting to the minimum width, all of the content start disappearing as shrinking continues.

      Priorities:
      1. content-width

          This is the default behavior of container which makes content fit the entire line.

      2. width

          If we set width, each item will conform with the set width. Assume we set width to 100px. on 4 items. When the container shrunk, it'll respect the set width until there is not enough overall width of the container. Which is 400px considering our example here. So if the container width get lesser than 400px, it'll adjust to the minimum-size of the content(longest words). It'll respect our set width when there is enough space for the container otherwise it'll adjust dynamically until minimum width of each item is attain.

          Note: At default, if we had nothing set on the container it'll look at the entire line when it's all on one line. Which will be the 'content-width' but if width is set it'll adjust to the width instead of the content-width. So, 'width' will be obey till will shrunk the container. Where it no longer fit everything as set width, then it starts shrinking the box.

      3. flex-basis

          This increase 'width' when container is at default(rows) But increase 'height' when 'flex-direction' is at column. 'flex-basis' overshadow' 'width' when both are set on an items. It'll adjust to set value of 'flex-basis' as long as there is enough room in the container, But as soon as we shrink the container down, just as before, it'll adjust to the 'minimum-width'

      4. min-width

          This looks at longest words to determine the minimum-width. Which determine how small each items can shrink to. For example, if 'flex-basis' is 200px and 'min-width' is 300px, the items will respect the 'min-width' instead of the 'flex-basis'. 'min-width' overshadow 'flex-basis' when it's larger than 'flex-basis'. Items does not shrink beyond 'min-width' size.

      5. max-width

          This looks at longest possible line of the text. So it's all of content lined up in one line. We can customize this, for example, if we set 'max-width' to 100px and our flex-basis is 200px. 'max-width' will overpower the flex-basis property because it's lower than the 'flex-basis'. 'max-width' is the maximum requirement for the items to grow. But if 'flex-basis' is smaller, it'll ignore the 'max-width'.

      Flex Reactivities
      • flex-grow
      • flex-shrink

        • CASES

        • flex-basis: 100px; flex-grow: 0; flex-shrink: 0

            They can be understood better by turning both off by setting them to '0'. So setting 'flex-basis' to 100px and the both grow & shrink to '0' make flexbox not a flexbox any longer as the container stops reacting to ability to increase and decrease.

        • flex-basis: 100px; flex-grow: 1; flex-shrink: 0

            Each items will start out as 100px, it'll then grow because we've switch the flex-grow on. And it'll respect that as long as there is enough space, it'll increase the width of each of the items until the entire width of the container is filled up by each items. So when the container width is reduce, it gets reduce to the flex shrink amount which is the 'flex-basis' value. Any further than that will makes the items to disappear.

        • flex-basis: 100px; flex-grow: 0; flex-shrink: 1 (Flexbox Default)

            By default, it'll set each items to the 'flex-basis' value but it'll not grow to fit the entire space of the container. But it'll shrink beyond the 'flex-basis' value. Until 'minimum width' is reach and because we did not set it, then it'll adjust to the 'content-width'

        • flex-basis: 100px; flex-grow: 1; flex-shrink: 1

            Switching on both means it'll take much width as possible and when shrink, it'll be as small as possible. Which 'flex-basis' is completely ignored. So here: 'flex-grow' represent 'max-width while' 'flex-shrink' represent 'min-width'. If these two are not set, it'll adjust to the content of those items.

        • flex-basis: auto; flex-grow: 1; flex-shrink: 1

            The default of 'flex-basis' is 'auto'. Which look at amount of content in each items, it'll gives more 'flex-basis' to the items with more content. And it'll give less width to the one with smaller content. If we want that to be equal width, we simply set it 'flex-basis' to '0'

          • SHORTHAND
          • flex: 1 1 0

              This means both flex-grow and flex-shrink is 1. flex-basis is '0'

          • flex: 1

              This means both flex-grow and flex-shrink is 1 and flex-basis is 0. Which is same as the first instance. The value/ratio can be more than '1'.

  • Grid(Two-dimensional layout)

      • Grid is more suitable to create complex web layout. 'Flexbox' is best to align content along one dimensional line. So either 'x or y' axis. While 'Grid' is good to align content alone a two dimensional layout. So both 'x and y' axis. Both flexbox and grid can be mix together. We can as well create one dimensional Grid. But it behave differently. Grid behavior tends to line up on each Other on a column and row basis and the gap between items match. Whereas, flexbox act differently because it's flexible and tries to adapt and squish content so that everything is place on screen. So depending on the behavior we seek, whether to prioritize spacing out content and aligning them or we want everything to be boxy like a grid. Check Flexbox@Grid for details comparison

      Grid usage: .container {display: grid; grid-template-columns: 1fr 2fr; grid-template-rows: 1fr 1fr; gap: 10px } This simply means. Create two columns with one column as fraction of size of 1 and the other column twice the size. Then create 2 rows with both 1 fraction of size with 10px of space. Note: Default behavior of grid is to take as much full width, but only has as much height as it allows to fit the content.

  • Grid Sizing

      There are severals types: Check Grid-sizing

      We'll see preview of different kind of grid-sizing. We'll see live preview e.g 'fixed-size': which is not responsive to screen size.

    • Fixed-Size

        .grid-container { display: grid; grid-template-rows: 100px 200px; grid-template-columns: 400px 800px; }

        In this instance, we define rows and columns by giving them fixed sizes(px). So we have 2 by 2. Two rows of 100px high for the first row, 200px high for the second row. Two columns of 400px wide for the first one and 800px wide for the second column. Fixed-size grid are not responsive. That means that if we go ahead and change the width or height, it's not going to change at all. It's going to stay fixed to the specified size we set above. In addition to 'px', we can also use 'Rem' Which is still fixed because 'rem' is relative to the root of our html. If our body/html has a size specified and we change that size then the 'rem' size will go up and down with it. But this does not make it responsive to the screen size. Instead of specifying grid-template rows and grid-template columns we can shorten it to "grid-template: 100px 200px / 400px 800px". Rows comes first, then column follows.

    • Auto-Size

        We can also use 'auto' => .grid-container { display: grid; grid-template-rows: 100px auto; grid-template-columns: 200px auto; } Unlike before 'Auto' now add a bit of responsiveness. Each row will try to fit to 100% of the horizontal available space. In our case now we'll have a first 'column' as '200px' then the other one which is auto will cover the rest space available width. 'Rows' behave differently, it doesn't try to take 100% height. Instead, it tries to fit the content. In our case, first row is 100px, the second row which is auto will try to fit the content.

    • Fractional-Size

        This is achieve using ratio(fr) to define layout for sizes eg: .grid-container { display: grid; grid-template-rows: 1fr 2fr; grid-template-columns: 1fr 2fr; }

        The above illustration depict that the first row is 1fr and the second row is twice the size of the first row. Same logic for column. This will grow and as well shrink depending on the width available for the horizontal and also the content size available for the vertical. This is responsive type as it always maintain the figure size provided. Even if we add additional contents, it'll expand and still keep the size uniform to the provided figure ratio.

    • MinMax Size

        Sometimes we want to define how the screen is responsive especially on the horizontal. .grid-container { display: grid; grid-template-rows: 200px 400px; grid-template-columns: 200px minmax(400px, 800px); }

        We denotes two rows with first being 200px and the second as 400px. The first column is 200px but the second column has two instances with first condition applicable to when the screen has little space we infer 400px as minimum display and 800px wide when the screen has enough space. This set a limit. Because sometimes, we have some kind of contents that we don't want to be too narrow or too wide. So our grid now range from 400px to 800px. So a stop at 400px and stop at 800px for responsiveness strain.

    • Repeat

        This cut-off repetition .grid-container { display: grid; grid-template-rows: repeat(2, 200px); grid-template-columns: repeat(2, 100px); }

        This simply denote that we want two 200px rows. And two 100px column. Which is same as defining rows as '200px, 200px'. And column as '100px, 100px'.

        What if we define grid than amount of div's we have? for example, we have 4 items or div's and we define 2 by 3 grid. So two rows and three columns. If we only have 4 items, they're fitted into our grid from the top left to the bottom right. So that means once we've laid out the first row, it'll double back and continue along the second row.   • Grid Item Placement

      We first need to look at some grid placement terminologies :

    • Lines

        This separate our tracks. We have horizontal lines and vertical lines, these is known as the grid lines. We can only control the grid lines by using the gap property to specify its width or height. But we can't change its color, we cant put anything in the lines. They are lines for layout.

    • Tracks

        When we create and size our grid, we create rows and columns. These is known as tracks. We have rows tracks and columns tracks and we usually create and size these tracks using grid-template-rows and grid-template-columns[grid-template] for shortcut.

    • Cells

        Cells is being created within the intersection of tracks. Which is a small unit also known as grid cells. They are the smallest unit in a grid and can be of different sizes. So one can be wider than the other that is to occupy more columns, while other can have height than the other, which means to occupy more rows. We can use multiple cells to create a grid items. So we can either position our grid cell horizontally or vertically across grid cells

    • Container

        This is the parent container div that contain div's items

    • Items

        This are the parent container children.

      • <div class="container">
      <div class="item cowboy"> </div>
      <div class="item astronaut"> </div>
      <div class="item book"> </div>
      </div>
      .container {
      height: 100vh;
      display: grid;
      gap: 3rem;
      grid-template-rows: 1fr 1fr;
      grid-template-columns: 1fr 1fr 1.5fr;
      }
      Note: By default, a div is going to span the entire width of the window, but it'll only occupy as much vertical space as there is content. Hence, the reason why we set height viewport height to 100, then it's also going to take that container and make it stretch all the way down to the bottom of the window. So the above code, we already known we create tracks using both grid-template-rows and grid-template-columns. So we created 3 columns with two of them being equal in width while the third one is a bit wider and we created two rows of equal height. So now, we've got tracks, lines and also container. Now the items will be place in by default. It'll start from the first and it'll go into each of the cells that are available. We can use Flexbox to target grid items to position them.

  • Overlay of items onto other items

      This is the biggest difference between grid and flexbox. Grid let us overlay items onto other items. So we can position items on top of each other in a grid.

    • .cowboy {
      background-color: #00B9FF;
      grid-area: 1 / 1 / 2 / 3
      }
      .astronaut {
      background-color: #03989E;
      grid-area: 2 / 1 / 3 / 3
      }
      .book {
      background-color: #E5833180;
      grid-area: 1 / 2 / 3 / 4
      }

      • The '80' number in book color represent transparency rate. In Hex code, the first six character correspond to the color so the amount of red, green, and blue. Then we can add a value known as 'alpha' which denote how much transparency there is. 80 is about 50% of transparency that allows us to see through the orange and the blue and the green. So the book coordinate is also set to overlap, starting from row one, column two then end at row three and column four.
